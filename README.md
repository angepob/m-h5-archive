# m-h5-archive

一个用于归档和发布多个独立 HTML 页面的仓库。

## 项目简介

本仓库旨在收集、整理并托管一批独立的 HTML 页面。这些页面可能包含工具、演示、信息展示或小型应用，彼此之间没有强关联，各自独立运行。

通过将页面托管于此，可以方便地通过 GitHub Pages 或其他静态托管服务访问，适合用于存档、分享或快速原型展示。

## 目录结构

```
m-h5-archive/
├── index.html          # 入口页面，展示归档页面列表
├── pages/              # 存放各个独立 HTML 页面
│   ├── page1/         # 示例页面目录
│   │   └── index.html
│   ├── page2/
│   │   └── index.html
│   └── ...
└── assets/             # 公共资源（可选）
    ├── css/
    ├── js/
    └── images/
```

- **index.html**：仓库首页，可作为导航页面，列出所有归档页面及其简要说明。
- **pages/**：每个独立页面建议放在单独的子目录中，避免文件名冲突，方便管理。
- **assets/**：如果多个页面共享样式、脚本或图片，可统一放置于此。

## 页面归档说明

- 每个页面应是一个完整的、自包含的 HTML 文件（或包含其依赖资源）。
- 建议为每个页面添加简短的说明，记录其用途、来源或创建时间（可在 `index.html` 中体现）。
- 页面内容应合法合规，不包含恶意代码、侵权内容或敏感信息。
- 归档的页面可能来自不同时间、不同项目，风格和功能上不做统一要求。

## 维护说明

- 本仓库由维护者不定期更新，添加或移除页面。
- 欢迎通过 Issue 提出归档建议，但请确保内容符合开源社区规范。
- 如需提交新页面，可 Fork 本仓库后创建 Pull Request，建议附带页面说明。
- 仓库不保证所有页面长期可用或兼容所有浏览器，请自行测试。

## 使用方式

1. 克隆本仓库到本地。
2. 将需要归档的 HTML 文件放入 `pages/` 目录下，并更新 `index.html` 添加链接。
3. 推送到 GitHub 仓库，开启 GitHub Pages 功能即可在线访问。
